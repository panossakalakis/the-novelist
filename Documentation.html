Working with Templates

What are Templates?

Templates are pre-built markdown files that provide a consistent structure for creating new entries. The Novelist includes templates for characters and locations. Each template contains predefined fields and sections to ensure consistency across your vault.

How to Insert a New Template

1

Create a New File

Navigate to the appropriate folder where you want to create your new entry:

  • For characters, go to Characters > [Appropriate Category]
  • For locations, go to Locations or create a subfolder if needed

Create a new markdown file by right-clicking the folder and selecting "New note", or use Ctrl+N (Cmd+N on Mac).

2

Click the Templater Icon

Look for the Templater plugin icon in the left sidebar (it appears as a template symbol). Click it to open the template browser.

3

Search for Templates

Type "templates" in the search box. You will see both available templates:

  • Character Template - Use for creating new character profiles
  • City Template - Use for creating new location or world entries
4

Select Your Template

Click on the template you want to use. The template content will be automatically inserted into your new file.

5

Customize Your Entry

Fill in the frontmatter properties and content sections with your specific information. Replace all placeholder text with your actual data.

Tip: Quick Template Access

The Templater plugin icon in the left sidebar provides the fastest way to insert templates. You can also access it using a keyboard shortcut if configured in your Obsidian settings.

Adding and Managing Properties

What are Properties?

Properties (also called frontmatter) are metadata fields at the top of a file, enclosed in three dashes (---). They help organize and categorize your vault entries, making them searchable and sortable.

How to Add New Properties

1

Open Your File

Open any markdown file in your vault where you want to add new properties.

2

Click the Add Property Button

Look at the top of the file in the properties panel. You will see a "+ Add Property" button. Click it to create a new property.

3

Name Your Property

Type the name of your new property in the input field. For example, "Status", "Priority", or "Theme".

4

Add the Value

Click on the property to open its value field and enter the appropriate value for that property.

How to Edit Properties

1

Open the File

Open the file containing the property you want to edit.

2

Click on the Property

In the properties panel at the top of the file, click directly on the property value you want to change.

3

Edit the Value

The property will become editable. Change the value to what you need and press Enter to confirm.

How to Remove Properties

1

Open the File

Open the file containing the property you want to remove.

2

Find the Property

Locate the property in the properties panel at the top of the file.

3

Delete the Property

Click the X icon or delete button next to the property name to remove it from the file.

Linking Everything Together

Why Linking Matters

Linking connects your notes together, creating a web of interconnected information. This helps you see relationships between characters, locations, and plot elements at a glance. The graph view will show all your connections visually.

Creating Links in Obsidian

1

Type the Link Syntax

To create a link to another note, use the syntax [[Note Name]]. Simply type the opening brackets, the name of the note you want to link to, and closing brackets.

Example: [[Raven Martinez]] creates a link to the Raven Martinez character file.

2

Link to Existing Notes

Start typing and Obsidian will show you a list of matching notes. Select the one you want to link to from the dropdown menu, or continue typing the full name and press Enter.

3

Create Links While Creating New Notes

You can also link to a note that doesn't exist yet by using the same syntax. Obsidian will create a link, and the note will appear as "unlinked" until you create it. When you eventually create the file, the link becomes active.

Recommended Linking Practices

Link Characters to Locations

In your character files, add a "Locations" property that lists the main locations where the character appears:

Locations: [[Ashenfall]] · [[Meridian Building]] · [[Ashveil]]

Link Related Characters

Create relationships between characters in their "Relationship" property:

Relationship: [[Jason Chen]]
Family Members: [[Kyle]]

Link Locations to Connected Locations

In location files, add connected locations to show proximity or relationships:

Connections: [[Ashenfall]] · [[Meridian Building]]

Link to Chapter Mentions

When a character or location is mentioned in a chapter, create a link to that chapter in the character/location file. You can also do the reverse, linking from chapters to the characters/locations featured.

Using the Graph View

Open the graph view to see all your links visualized:

  1. Click the graph icon in the left sidebar, or press Ctrl+G (Cmd+G on Mac)
  2. Hover over nodes to see which files they connect to
  3. Click on a node to navigate to that file
  4. Use the settings to filter by file type or connection strength
Tip: Consistent Link Formatting

For links in properties, use the format [[Note Name]] without extra spaces. Separate multiple links with · or another delimiter for readability.

Usage Guides

Creating Your First Character

  1. Go to Templates and copy the Character Template
  2. Navigate to Characters and then to the appropriate subfolder (Lead, Important, or Non-Important)
  3. Create a new file with your character's name
  4. Paste the template and fill in the frontmatter properties
  5. Complete each section: Basic Information, Personality, and Backstory
  6. Link this character to other characters and locations they interact with
  7. Save the file

Creating Your First Location

  1. Go to Templates and copy the City Template
  2. Navigate to the Locations folder
  3. Create a new subfolder for related locations if needed (e.g., create an "Ashenfall" folder for city districts)
  4. Create a new file with your location's name
  5. Paste the template and update the frontmatter with location details
  6. Write the Overview and Description sections
  7. Link to characters who live in or visit this location
  8. Save the file

Organizing Chapters

Chapters are stored in the Chapters folder. Use the All Chapters.md file as your table of contents:

  1. Create individual files for each chapter (Chapter 1.md, Chapter 2.md, etc.)
  2. In All Chapters.md, maintain a master list of all chapters with links
  3. Include a brief summary or important notes for each chapter
  4. Link characters and locations featured in each chapter

Using the Worldbuilding Canvas

The Worldbuilding.canvas file provides a visual layout of your world:

  1. Open the Worldbuilding.canvas file to see the visual layout
  2. You can rearrange elements by drag and drop
  3. Click on elements to jump directly to those notes
  4. Add new elements by right-clicking on the canvas
  5. This is useful for planning and seeing relationships at a glance

Tracking Writing Goals

The Novelist includes writing goal tracking:

  1. Right-click on any folder or file in the Chapters section
  2. Select "Add writing goal"
  3. Set your daily word count target
  4. Track your progress in the left panel
  5. The system records your daily word count automatically

Managing Ideas and Plot Points

Use the Notes folder to store creative ideas:

  • Ending Ideas.md - Brainstorm possible endings
  • Plot Twist Ideas.md - Store unexpected turns and revelations
  • Trilogy Ideas.md - Plan connections between multiple books

Link these ideas to the relevant characters and locations they affect.

Marketing and Planning

In the Notes > Marketing folder, you'll find:

  • 4 Phases Marketing Plan.md - A structured approach to marketing your book
  • Build a Website.md - Guide for creating an author website

Customizing the Theme

To match your personal style:

  1. Go to Obsidian Settings
  2. Select "Appearance"
  3. Choose your preferred theme (light, dark, etc.)
  4. For advanced customization, go to Settings > Style Settings > AnuPpuccin
  5. Adjust colors and typography to your preference

Syncing Your Vault

To keep your vault updated across devices:

  1. Install Obsidian Sync or a third-party plugin (Remotely Save or Google Drive Sync)
  2. Configure your preferred sync method in Obsidian settings
  3. Enable auto-save if desired
  4. Your vault will now sync automatically across your devices

Frequently Asked Questions

How do I rename a file without breaking links? v

Obsidian automatically updates all linked references when you rename a file. Simply right-click the file, select "Rename", and update the filename. All links throughout your vault will be updated automatically.

Can I add my own custom template? v

Yes. Create a new markdown file in the Extras > Templates folder with your custom structure. You can then use it the same way as the built-in templates. If you want to use Obsidian's template plugin for quick insertion, enable it in Settings and set the template folder location.

How do I search across the entire vault? v

Use Ctrl+Shift+F (Cmd+Shift+F on Mac) to open the global search. Type the term you want to search for, and Obsidian will show all matching files and snippets of the text around the matches.

Can I add images to my character/location files? v

Yes. The templates include placeholder text "Image goes here..." where you should add images. To add an image, place the image file in the Media folder, then use the syntax ![[Image Name.png]] to embed it in your markdown. Replace the placeholder text with this syntax.

What is the difference between internal and external links? v

Internal links (using [[Note Name]]) only work within your vault and link to other notes inside it. External links (using standard markdown [text](url)) link to websites or outside resources. The Novelist primarily uses internal links to connect your story elements.

How do I see all files that link to a specific note? v

Open any note and look at the right sidebar. Under "Backlinks", you'll see all other notes that link to the current file. This is useful for tracking how many characters interact with a location or how many times a plot point is referenced.

Can I export my vault for publishing? v

Obsidian can export files to various formats. Use the export feature in Obsidian, or install the Vellum app (mentioned in the instructions) to format your book for publication. You can export individual chapters as markdown or PDF and import them into professional publishing software.

How do I track character arcs across chapters? v

In each character file, use the "Locations" property to list chapters where they appear. You can also create a section in the chapter file listing which characters appear. Link these references together. The graph view can then show you a visual representation of character presence across chapters.

What should I do if I accidentally delete a file? v

Deleted files are moved to the system trash. Go to your computer's trash/recycle bin and restore the file from there. If you're using Obsidian Sync or another syncing service, check the sync provider's trash or recovery options.

Can I create nested subfolders for better organization? v

Yes. You can create folders and subfolders to any depth. For example, create Locations > Ashenfall > Districts or Characters > Antagonists > Royal Family. References still work as long as the filename is unique, but deep nesting can make navigation more complex.

How do I use the Worldbuilding canvas effectively? v

Open the Worldbuilding.canvas file to see your world organized visually. You can drag elements around to arrange them logically. Links show relationships between elements at a glance. Use this for planning story structure, character relationships, and location layout before diving into detailed writing.

What is the best way to organize character backstories? v

The Character Template includes a Backstory section with subsections for Basics and Main Backstory. Use the Basics subsection for key facts and memories. Use the Main Backstory section for a narrative account of their life before the story begins. This is the approach used in the example character files.

How often should I save my vault? v

Obsidian auto-saves files immediately as you type. If you have syncing enabled, changes sync automatically. You don't need to manually save. However, if you're using external sync services, make sure they're configured correctly so your data stays backed up across devices.
